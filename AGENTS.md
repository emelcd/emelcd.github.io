## Content rules (critical)

1. **Never edit site copy in this repo.** Edit `data/cv.json` in the backend.
2. `src/lib/content-data.json` is a **build-time fallback** regenerated by
   `bun run fetch-content`. Local edits get overwritten.
3. **No hardcoded UI copy** in components — read from `usePreferences().t`
   (localized `Content`). Exceptions: terminal decoration, ThemeMenu/nav chrome
   labels that live in the component (ES/EN local copy objects).
4. Content loads twice by design: baked JSON first, then runtime `fetchContent()`
   so backend edits go live without a frontend rebuild.
5. When adding Content fields, update: `Content` type + `normalizePortfolio` in
   `content.ts`, both `es`/`en` in backend `cv.json`, and sync the baked
   fallback (or run `fetch-content` after backend deploy).

## Theming

Visitor prefs (persisted in `localStorage` key `emelcd-prefs`):

- language (`es` / `en`)
- dark / light
- accent (`ACCENTS` / `ACCENT_ORDER` in `content.ts`)
- font (`FONTS` / `FONT_ORDER` in `content.ts`)

CSS wiring:

- Accent → `--accent-400/600/800` on `<html>`
- Font → **`--font-ui`** on `<html>` (not `--font-sans` directly)
- Tailwind `@theme` maps `--font-sans` / `--font-heading` → `var(--font-ui)`
- FOUC prevention: inline script in `index.html` applies stored prefs before paint

**Do not** set `--font-sans` alone to change fonts — `@theme inline` inlines
Geist unless you go through `--font-ui`.

Navbar chrome:

- One preferences control: `ThemeMenu` (SlidersHorizontal icon). Keep language,
  theme, color, and font **inside that single menu**.
- Mobile/tablet: hamburger → dialog with numbered section links.
- Desktop (`lg+`): inline nav links + `SectionDots` rail; both use scroll-spy.

## Mobile / responsive

Target ~375px without horizontal scroll.

- Keep `overflow-x: clip` on `html`/`body` and the app root.
- Prefer `min-w-0`, `break-words`, stacked flex (`flex-col sm:flex-row`) over
  rigid `shrink-0` rows.
- Large decorative glows must be clipped by a parent (`overflow-hidden`) or use
  `min()` / `vw` caps — never fixed widths like `w-[900px]` unclipped.
- ThemeMenu popup: `max-h` + `overflow-y-auto` for short viewports.
- Terminal output: `break-words`; game boards: `overflow-x-auto`.

## UI conventions

- Functional components, `@/` imports, Tailwind utilities, base-ui primitives.
- Match existing visual language; don’t invent a parallel design system.
- Prefer extending `ThemeMenu` / `preferences.tsx` / `Navbar` over new panels.
- Section indices in headers should stay sequential if sections are added/removed.
